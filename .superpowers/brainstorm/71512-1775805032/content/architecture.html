Yeek Session Manager - 三层架构

系统架构图
Interface Layer（界面层）
┌─────────────┐ ┌──────────────┐
│ CLI (Clap)   │ │ TUI (Ratatui) │
└──────┬──────┘ └──────┬───────┘
       │           │
Core Engine（核心引擎）
┌──────────────────────────────┐
│ SessionManager            │
│ ├─ CRUD operations         │
│ ├─ Search (FTS5)          │
│ ├─ Tag / Filter           │
│ └─ Conversion orchestrator   │
└──────────────┬───────────────┘
Data & Adapter Layer（数据层）
┌────────────┐ ┌─────────────────────┐
│ SQLite    │ │ Adapter Registry   │
│ (rusqlite) │ │ ├─ ClaudeCodeAdapter │
│           │ │ ├─ CodexAdapter    │
│           │ │ └─ GenericAdapter   │
└────────────┘ └─────────────────────┘
数据流

Import 流程：
Agent 原始文件
→ Adapter.parse()
→ 统一 Session 模型
→ SQLite 存储

Export 流程：
SQLite 查询
→ 统一 Session 模型
→ Adapter.export()
→ 目标 Agent 格式

Resume 流程：
SQLite 查询
→ Adapter.resume()
→ 启动目标 Agent 进程
→ 注入 session 上下文

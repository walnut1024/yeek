Adapter Trait 设计

每个 agent 实现这套接口 — 新增 agent 只需实现一个 struct

Rust Trait 定义

/// Agent 适配器 trait — 所有 agent 必须实现
pub trait AgentAdapter: Send + Sync {
  /// 适配器名称（如 "claude_code", "codex"）
  fn name(&self) -> &str;

  /// 扫描 agent 的 session 存储目录，返回发现的 session 列表
  /// 返回 (source_id, file_path, mtime) 的列表
  async fn discover_sessions(&self) -> Result<Vec<DiscoveredSession>>;

  /// 解析单个 session 文件
  /// offset: 从第 N 行开始解析（支持增量）
  /// 返回解析出的统一消息 + 新的 offset
  fn parse_session(
    &self,
    path: &Path,
    offset: u64,
    ) -> Result<ParsedSession>;

  /// 将统一格式导出为 agent 的原生格式
  fn export_session(
    &self,
    session: &Session,
    messages: &[Message],
    target_path: &Path,
  ) -> Result<()>;

  /// 将 session 上下文注入到 agent 并启动 resume
  async fn resume_session(
    &self,
    session: &Session,
    messages: &[Message],
  ) -> Result<()>;

  /// 判断该 session 当前是否仍在活跃运行
  fn is_active(&self, source_id: &str) -> bool;
}

具体实现示例

ClaudeCodeAdapter

  • discover_sessions → 扫描 ~/.claude/projects/*/
  • parse_session → 解析 JSONL，提取 parentUuid 链、role、content
  • export_session → 重建 JSONL，还原 parentUuid 链结构
  • resume_session → 写入 session 文件 + 执行 claude --resume
  • is_active → 检查 ~/.claude/sessions/<PID>.json

CodexAdapter

  • discover_sessions → 读取 state_5.sqlite threads 表
  • parse_session → 解析 rollout JSONL，转换 4 种事件类型
  • export_session → 生成 rollout JSONL + 写入 threads 表
  • resume_session → 执行 codex resume --id
  • is_active → 查询 state_5.sqlite 中未完成的 thread
